# 第8章 離散時間シミュレーションとフィードバック制御

第7章での微分方程式の計算、素晴らしかったです！`ode45`は非常に強力ですが、実際のロボットやドローンに搭載されているマイコンは、連続的な時間ではなく「一定の周期（サンプリング時間）」ごとに計算を行っています。

第8章では、マイコンの動作に近い「離散時間シミュレーション」と、制御工学の基礎である「フィードバック制御」をMATLABで実装する方法を学びます。

## 1. データの事前割り当て（zeros関数）
`for`文を使って配列（ベクトル）の要素を1つずつ計算して追加していく場合、あらかじめ配列の「枠」を作っておくと計算速度が劇的に向上します。MATLABでは `zeros` 関数を使って、すべての要素が $0$ の行列やベクトルを作成します。

```matlab
% 要素が100個ある行ベクトル（1行100列）を0で初期化
v = zeros(1, 100);

% 3行3列の0行列を作成
A = zeros(3, 3);
```

## 2. オイラー法（Euler Method）による状態更新
`ode45`を使わずに、自分自身で微分方程式を解く最もシンプルな手法が「オイラー法」です。微小な時間ステップ $\Delta t$ の間の変化率は一定であると仮定し、次のステップの状態を計算します。

現在の状態を $y_k$、その時の微分（傾き）を $\frac{dy}{dt}$ とすると、次の状態 $y_{k+1}$ は以下の近似式で求まります。

$$y_{k+1} = y_k + \frac{dy}{dt} \Delta t$$

## 3. フィードバック制御（P制御）の概念
ロボットを希望の動きにするためには、現在の状態をセンサーで読み取り、目標値との「誤差」に応じて操作量を決める必要があります。これをフィードバック制御と呼びます。

最も基本的な「比例制御（P制御）」では、目標値 $r$ と現在の状態 $y$ の誤差 $e$ を計算し、それに比例ゲイン $K_p$ を掛けたものを操作量（モーターの推力など） $u$ とします。

$$e = r - y$$
$$u = K_p e$$

## 例題：ドローンの水平方向の速度制御（P制御）
第7章で登場したドローンが、今度は水平方向に飛行しているとします。ドローンの質量 $m = 1.2$ [kg]、空気抵抗係数 $c = 0.5$ [N$\cdot$s/m] とします。
目標とする水平速度 $v_{ref} = 5.0$ [m/s] を維持するために、P制御を用いてモーターの推進力 $u$ [N] を決定します。

水平方向の運動方程式は以下の通りです。
$$m \frac{dv}{dt} = u - cv$$

以下の条件を満たすMATLABコードを作成し、速度の推移をグラフ化してください。
1. サンプリング時間 $dt = 0.05$ [s] とし、シミュレーション時間は $0$秒から $10$秒までとする。（時間ベクトル `t` を作成する）
2. 配列のサイズを事前に取得し、速度ベクトル `v` と推力ベクトル `u` を `zeros` で初期化する。（初期速度は $0$ とする）
3. 比例ゲイン $K_p = 2.0$ とする。
4. `for`ループを用いて、各ステップ $i$ において以下の順に計算を行う。
   * 誤差の計算: $e = v_{ref} - v_i$
   * 推力の計算: $u_i = K_p e$
   * 加速度の計算: $\frac{dv}{dt} = \frac{1}{m} (u_i - c v_i)$
   * オイラー法による速度の更新: $v_{i+1} = v_i + \frac{dv}{dt} dt$
5. 時間 `t` に対して、速度 `v` のグラフを描画する。

## 例題の解答

```matlab
% 物理パラメータ
m = 1.2;      % 質量 [kg]
c = 0.5;      % 空気抵抗係数 [N*s/m]

% 制御パラメータ
v_ref = 5.0;  % 目標速度 [m/s]
Kp = 2.0;     % 比例ゲイン

% 時間の設定
dt = 0.05;          % サンプリング時間 [s]
t = 0:dt:10;        % 0から10秒までdt刻みの時間ベクトル
N = length(t);      % データ点の総数

% ベクトルの事前割り当て（メモリ確保）
v = zeros(1, N);    % 速度ベクトル
u = zeros(1, N);    % 推力ベクトル

% 離散時間シミュレーション (オイラー法)
% v(N+1) を計算しないように、ループは N-1 までとします
for i = 1:(N-1)
    % 1. 現在の誤差を計算
    e = v_ref - v(i);
    
    % 2. P制御による推力（操作量）の決定
    u(i) = Kp * e;
    
    % 3. 運動方程式から加速度を計算
    dvdt = (u(i) - c * v(i)) / m;
    
    % 4. オイラー法で次のステップの速度を計算
    v(i+1) = v(i) + dvdt * dt;
end

% 最後のステップの推力を計算（グラフ描画用）
u(N) = Kp * (v_ref - v(N));

% グラフの描画
plot(t, v, 'LineWidth', 2);
hold on;
plot(t, v_ref * ones(1, N), '--r', 'LineWidth', 1.5); % 目標値の破線
hold off;

xlabel('Time [s]');
ylabel('Velocity [m/s]');
title('Drone Velocity Control (P-Control)');
legend('Current Velocity', 'Target Velocity', 'Location', 'southeast');
grid on;
```

### 解答コードの解説
* `N = length(t);` : `length` 関数を使うと、ベクトルの要素数（長さ）を取得できます。この `N` を使って `for` ループの回数を設定します。
* ループの範囲を `1:(N-1)` としているのがポイントです。ループの中で `v(i+1)` を計算しているため、もし `i` が `N` まで回ってしまうと、配列のサイズをはみ出して `v(N+1)` にアクセスしようとしてエラーになります。
* `plot` 関数で `v_ref * ones(1, N)` を使っています。`ones` 関数は全ての要素が $1$ のベクトルを作るため、これに目標値を掛けることで、常に $5.0$ を示す水平な補助線（破線 `--r`）を描画しています。

このように `for` ループとオイラー法を使うことで、制御アルゴリズム（PID制御など）を1ステップずつ自分の手で組み込み、シミュレーションすることが可能になります。これはROS2などを用いたC++やPythonでの制御実装の基礎となる考え方です。
